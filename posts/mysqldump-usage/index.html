mysqldump 用法

Posted at — Mar 16, 2018

导出MySQL数据库和表的结构和内容的工具

用法:

mysqldump [OPTIONS] database [tables]
mysqldump [OPTIONS] --databases [OPTIONS] DB1 [DB2 DB3...]
mysqldump [OPTIONS] --all-databases [OPTIONS]

默认选项按给定顺序从以下文件中读取：

/etc/my.cnf /etc/mysql/my.cnf ~/.my.cnf

常用选项：

选项 说明
-A, –all-databases 导出所有的数据库，同 --databases
-Y, –all-tablespaces 导出所有的表空间
-B, –databases 导出指定的数据库，如果没有给数据库的名字，默认会导出所有的数据库，同--all-databases
–default-character-set=name 设置默认字符集。
-E, –events 导出事件
-e, –extended-insert 导出的SQL使用多行 insert values列表
-f, –force 忽略错误，强制导出
-h, –host=name 连接数据库的地址(或者 IP)
–ignore-table 指定不导出的表，此命令不能指定多个表，但是可以通过多次使用此选项来指定多个不需要导出的表，表格需要指定字段库饭名称，如：--ignore-table=database.table
–insert-ignore 使用 insert ignore 代码 insert
-x, –lock-all-tables 导出的时候锁定所有数据库的表
-n, –no-create-db 禁止 CREATE DATABASE ... IF EXISTS语句，
–log-error=name 将警告和错误附加到给定文件
-t, –no-create-info 不导出表格创建等信息。
-d, –no-data 不导出表格数据
-p, –password[=name] 连接到服务器的密码
-P, –port=# 连接服务器的端口
-q, –quick 不缓冲查询，直接导出
-r, –result-file=name 直接输出到给定的文件,
–dump-date 把导出的日期输出到结尾
-u, –user=name 连接数据库的登录名
–tables 多个表，此选项会覆盖–databases

实例用法

1.导出整个库（一个或者多个）

mysqldump -u yourusername -p yourpassword --databases yourdatabase1 yourdatabase2 #导出N个库

mysqldump -u yourusername -p yourpassword --all-databases #导出所有的数据库
  1. 只导出指定表的数据
mysqldump -t youdatabase -u yourusername -p yourpassword --tables table_name1 tablename2
  1. 只导出指定表的结构
mysqldump -d youdatabase -u yourusername -p yourpassword --tables table_name1 tablename2
  1. 同时导出表数据和结构
mysqldump youdatabase -u yourusername -p yourpassword  --tables table_name1 tablename2
  1. 不导出某些表
mysqldump -d youdatabase -u yourusername -p yourpassword --ignore-table=youdatabase.yourtable1 --ignore-table=youdatabase.yourtable2
